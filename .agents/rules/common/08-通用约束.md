# 通用约束

## 语言规范（NON-NEGOTIABLE）

- 所有文档和代码注释必须使用中文
- 代码变量/函数命名仍使用英文（遵循编码规范）

## 可观测性与兜底

### 通用规范

- 错误处理必须有统一机制（错误边界 / 全局异常捕获），禁止吞掉异常不处理
- 生产环境禁止残留 `console.log`、`console.debug`；仅允许 `console.warn` 和 `console.error` 用于真实告警
- 需要调试日志时，使用项目统一的 logger 工具，支持按环境区分日志级别

### AI 业务场景

以下约束适用于包含 AI 能力的业务模块：

- AI 任务需要完整链路：任务 ID、素材/模型引用、日志追踪
- 失败时提供兜底提示与重试机制

## 依赖管理

- 新增第三方依赖前需评估 bundle size 影响，优先使用项目已有工具库
- 锁定包管理器（pnpm），禁止混用 npm / yarn
- 禁止引入超过 12 个月无更新且无明确维护者的依赖
- 依赖版本通过 lock 文件锁定，禁止使用 `*` 或 `latest` 版本号

## 安全约束

- 禁止在代码中硬编码密钥、token、密码等敏感信息
- 敏感配置通过 `.env` 文件管理，`.env` 文件必须加入 `.gitignore`
- 用户输入在拼接 HTML 或 URL 前必须转义，防止 XSS
- 第三方脚本（统计、广告等）需评估安全风险后再引入

## 调试代码约束

- 提交代码前必须清除 `debugger` 语句
- 禁止提交被注释掉的大段代码块（超过 5 行），废弃代码应直接删除
- 临时调试用的 `// @ts-ignore`、`// eslint-disable` 必须附带原因注释，合并前清除

## Git 提交规范

- Commit message 遵循 [Conventional Commits](https://www.conventionalcommits.org/) 格式：`<type>(<scope>): <description>`
- 常用 type：`feat`、`fix`、`refactor`、`style`、`docs`、`test`、`chore`
- 单次提交聚焦一个变更目的，禁止在一个 commit 中混合不相关改动

## 占位元素使用规范

### 使用场景

以下情况使用占位元素，不实现临时方案：

- 图标资源未确定（等待 iconfont）
- 图片资源未确定（等待设计提供）
- 第三方资源未接入

### 处理原则

| 场景 | 处理方式 |
|------|----------|
| 图标 | 使用 CSS Modules 占位 class（如 `placeholderIcon`） |
| 图片 | 使用 CSS Modules 占位 class（如 `placeholderImage`）或组件库骨架屏组件 |
| 后续替换 | 添加 `TODO` 注释标记替换点 |

### 示例

**React（TSX）**：

```tsx
{/* 工艺单卡片图片 */}
<div className={styles.placeholderImage} />

{/* 新建按钮图标（等待 iconfont） */}
<div className={styles.placeholderIcon}>
  {/* TODO: 替换为 iconfont 图标 */}
</div>
```

**Vue（SFC template）**：

```vue
<!-- 工艺单卡片图片 -->
<div :class="$style.placeholderImage" />

<!-- 新建按钮图标（等待 iconfont） -->
<div :class="$style.placeholderIcon">
  <!-- TODO: 替换为 iconfont 图标 -->
</div>
```

### 约束

- 禁止自行绘制临时替代图标（包括内联 SVG、canvas 等）
- 禁止使用外部 placeholder 图片服务
- 允许使用组件库提供的骨架屏组件（Ant Design `Skeleton`、Element Plus `ElSkeleton`）
- 明确标记替换点，便于后续批量替换
