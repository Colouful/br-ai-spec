# 项目结构（NON-NEGOTIABLE）

## 目录结构

```
src/
├── api/           # 接口封装，按业务模块划分
├── assets/        # 字体、图标、静态图片等资源
├── components/    # 可复用 UI 组件
├── composables/   # 组合式函数（useXxx）
├── constants/     # 业务常量、枚举
├── layouts/       # 布局组件（DefaultLayout、BlankLayout 等）
├── router/        # 路由配置与导航守卫
├── stores/        # Pinia store，按业务拆分
├── styles/        # 全局样式、变量、mixins
├── types/         # 全局类型声明与 DTO
├── utils/         # 通用工具函数
├── views/         # 页面视图
├── App.vue        # 根组件
└── main.ts        # 入口文件
```

## 结构约束（禁止在 src 下新建非标准目录）

| 类型 | 目录 | 规范 |
|------|------|------|
| 接口 | `src/api/` | 按模块划分，每个模块含请求函数与类型定义 |
| 状态 | `src/stores/` | 按业务模块划分，暴露 `useXxxStore` |
| 工具 | `src/utils/` | 按功能模块划分，统一导出 |
| 类型 | `src/types/` | 全局共享类型，模块私有类型就近放置 |
| 组件 | `src/components/` | 每个组件独立目录，包含 `index.vue` |
| 页面 | `src/views/` | 每个页面独立目录，使用 `kebab-case` 命名 |
| 组合式函数 | `src/composables/` | `useXxx.ts` 命名，纯逻辑复用 |

## 页面目录内的组件放置规则

页面专用组件放在 `src/views/<page>/components/` 目录：

```
src/views/<page>/
  ├─ components/           # 仅供当前页面使用的子组件
  │   └─ SomeWidget/
  │       ├─ index.vue
  │       └─ types.ts
  ├─ index.vue             # 页面主组件
  └─ style.module.scss     # 页面样式（可选）
```

**判断标准**：

- 通用组件：多处复用、业务无关 → `src/components/`
- 页面级组件：仅单页使用 → 页面目录下的 `components/`

**约束**：`src/components/` 只存放通用组件，禁止将仅单页使用的组件放入。

## Mock 数据策略

当后端接口未完成时：

1. 在 `src/api/<module>.ts` 中编写 mock 函数返回模拟数据
2. 类型放在 `src/types/<module>.ts`（单文件），不创建目录
3. 不创建独立 Pinia store，必要时使用组件内 `ref`/`reactive`
4. 禁止创建复杂 mock 结构或预留过多扩展字段
